## Dual-Source Mode Operation

The `convert_verification_to_tpdg.py` script was executed with these flags:

```bash
python3 scripts/convert_verification_to_tpdg.py \
    --test-case-dir test-acceptance/test_cases \
    --logs-dir test-acceptance/execution_logs \
    --recursive \
    --output test-acceptance/results/acceptance_test_results_container.yaml \
    --title "Acceptance Test Suite Results" \
    --project "Test Case Manager - Acceptance Test Suite" \
    --verbose
```

**Dual-source mode** means the script:
1. Scans the test case directory for YAML files with `type: test_case`
2. For each test case, looks for a matching execution log in the logs directory
3. Builds verification results by combining:
   - Test case metadata (from YAML)
   - Execution results (from JSON logs)
4. Converts the combined data to TPDG container format

**Processing Details**:
- Found 89 YAML files in `test-acceptance/test_cases/`
- Identified 76 valid test cases (files with `type: test_case`)
- Attempted to match each with execution logs
- All execution logs were missing (expected - no tests run yet)
- Generated container with all steps marked as `NotExecuted`
