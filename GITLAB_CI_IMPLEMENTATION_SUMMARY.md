- Simulates complete GitLab CI pipeline execution
- Verifies end-to-end workflow

### GitLab CI Simulation Script (`simulate-gitlab-ci-pages.sh`)

Provides local dry-run capability:

#### Features
- **Step 1**: Prerequisites check (Docker, required files)
- **Step 2**: Pull python:3.11 Docker image
- **Step 3**: Prepare working directory
- **Step 4**: Install dependencies (pip install)
- **Step 5**: Build documentation (mkdocs build)
- **Step 6**: Verify artifacts
- **Step 7**: Copy artifacts to project root
- **Step 8**: Display artifact configuration info
- **Step 9**: Display cache configuration info

#### Command Options
- `--clean`: Remove existing public/ directory before simulation
- `--keep-temp`: Preserve temporary files for debugging
- `--verbose`: Enable detailed output
- `--help`: Show usage information

## Technical Details

### Shell Script Compatibility
- **Bash version**: 3.2+ compatible (macOS default)
- **Logger library**: Uses centralized logging (`scripts/lib/logger.sh`)
- **Cleanup**: Automatic temp directory cleanup
- **Error handling**: Proper exit codes and error messages

### Docker Integration
- **Image**: python:3.11
- **Isolation**: Runs in isolated containers
- **Volume mounts**: Temporary directory mounting
- **Cleanup**: No persistent state

### Test Automation
- **Exit codes**: 0 for success, 1 for failure
- **Output format**: Color-coded with pass/fail indicators

- **Counters**: Tracks passed/failed tests
